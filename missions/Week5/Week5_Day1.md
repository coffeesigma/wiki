### 리뷰

#### 오늘 한 작업

- 코드 리뷰 품질 향상을 위한 PR 리뷰 우선순위 규칙 추가 (P1~P3로 구분)
    
- 작업 관리 효율화를 위해 WBS 작성 시도
    
- ERD 도메인 기반 업무 분담 및 개발 착수
    
- Spring Security 없이 인증/인가 로직 구현
    
    - Redis 기반 Refresh Token 및 Blacklist 구현
        
    - JWT 토큰 발급 및 관리 로직 구현
        
    - 커스텀 암호화, 인증 로직 및 어노테이션 기반의 권한 관리 시스템 구축
        
- 로컬 개발 환경 세팅 (Redis, MySQL Docker Compose 구성 및 application-dev.yml 설정)
    

#### 작업 과정에서의 배운 것

- Spring Security 없이 직접 인증/인가 시스템을 구현해보니 JWT, Refresh Token, 그리고 보안 관련 로직에 대해 깊게 이해할 수 있었으며, 실제 적용 가능한 기술을 습득했다.
    
- 로컬 개발 환경을 구성하면서 팀원들의 개발 생산성을 높이는 환경 세팅이 프로젝트 전반의 효율성에 큰 영향을 준다는 점을 배웠다.
    

#### 어려웠던 점

- WBS를 통해 세부 작업 관리를 시도했지만, 스프린트 일정이 타이트한 상황에서 추가적인 관리 오버헤드가 너무 크다는 판단을 하여 사용하지 않기로 결정하는 과정이 어려웠다.
    
- 인증/인가 로직을 구현하며 JWT 토큰 관리, 커스텀 암호화 등 예상하지 못한 세부 기술 문제들을 다루느라 많은 시간이 소요되었다.
    

### 회고 (KPT)

#### Keep

- 장기적인 개발 생산성을 고려해 프로젝트 초반부터 인증/인가 로직과 로컬 개발 환경을 철저히 준비한 점이 좋았다.
    
- 코드 리뷰의 질을 높이기 위한 명확한 리뷰 우선순위 컨벤션을 도입하여 팀 내 코드 퀄리티에 긍정적인 영향을 줄 것으로 기대된다.
    

#### Problem

- 인증/인가 작업의 우선순위를 조정하지 못하고 하루만에 너무 많은 기능을 완성하려고 했던 점이 욕심이었던 것 같다.
    

#### Try

- 중요한 기능을 하루에 전부 구현하려 하지 말고, 단계적으로 명확한 우선순위를 두고 진행하자.
